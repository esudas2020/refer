Installation Guide for PyRx in BricsCAD V25

To successfully get the PYLOAD command working with PyRx in a fresh BricsCAD V25 system, follow these exact steps to ensure all dynamic dependencies are satisfied:

Step 1: Install Python 3.12

PyRx specifically requires Python 3.12 (Python 3.13 or newer will cause the module load to fail). 1. Download Python 3.12.x from python.org. 2. During the installation window, make sure to check the box “Add Python to PATH”. 3. Complete the installation.

Step 2: Install PyRx Package

  1. Open up a Command Prompt.

  2. Run the pip installation command:

    python -m pip install cad-pyrx

    (This will download PyRx and its graphical dependency wrapper, wxPython).

Step 3: Copy Missing DLL Dependencies

By default, the PyRx C++ plugin (PyRxV25.0.brx) requires the core Python 3.12 DLLs and the wxPython GUI DLLs to be located directly inside its own folder to securely initialize within the BricsCAD memory space. 1. Find your Python folder (e.g., C:\Program Files\Python312\). Copy python3.dll and python312.dll. 2. Find the PyRx folder where pip installed it. You can typically find it at %AppData%\Python\Python312\site-packages\pyrx. Paste the two Python DLLs here. 3. Find the wx folder (e.g., %AppData%\Python\Python312\site-packages\wx). 4. Copy all .dll files from inside the wx folder. 5. Paste all the wx DLLs right next to the pyrx files inside the pyrx folder.

Step 4: Load PyRx into BricsCAD

  1. Launch BricsCAD V25.
  2. Type APPLOAD in the command line and press Enter.
  3. Browse to the pyrx folder where you pasted the DLLs from Step 3.
  4. Select the RxLoaderV25.0.brx file and load it.
  5. You should see a success message. Type PYLOAD and hit Enter. The command will now be recognized!
